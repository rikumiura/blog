
# ブログサイト SUDOモデリング定義書

## 1. S：システム関連図 (System Context Diagram)
システムの境界と外部との関わりを定義します。

*   **アクター**
    *   **執筆者**: あなた一人（管理者兼任）。
*   **外部システム**
    *   **Zenn**: APIを通じて記事の投稿・同期を行う（将来連携）。
    *   **画像ストレージ**: 記事内で使用する画像を管理・保存する。
*   **システム境界（スコープ）**
    *   ローカルサイトへの記事公開。
    *   Zennへの自動投稿・同期機能は現時点では含まない（将来拡張）。

## 2. U：ユースケース図 (Use Case Diagram)
ユーザーの要求に対するシステムの振る舞いと、現在の注力範囲を定義します。

*   **優先ユースケース（今回のターゲット）**
    *   記事を執筆する（新規作成・編集）。
    *   記事を**ローカルサイトに公開**する（最優先事項）。
*   **今後の拡張ユースケース**
    *   公開済み記事をZennへ投稿（同期）する。
    *   外部サイト（Zenn）での公開URLを管理する。
*   **対象外（現時点）**
    *   読者からのコメント管理。
    *   SNSへの更新通知。

## 3. D & O：ドメインモデル図 & オブジェクト図
ビジネスルールと具体的なデータの姿を定義します。

### ドメインモデル図（責務の整理）
*   **主要エンティティ**: `Article`（記事）
*   **この文書の責務**: システム境界・ユースケース・用語の定義。
*   **ドメイン詳細の正本**: [`aggregation.md`](./aggregation.md)
    *   `Article` 集約の構成要素
    *   不変条件（タイトル必須・状態遷移・公開日時など）
    *   `Body` と Cloudflare R2 の保存責務分離

## 4. ユビキタス言語（対訳辞書）
実装時の表記揺れを防ぐための用語定義です。

| 日本語名 | 英語名（コード上の表記） |
| :--- | :--- |
| 記事 | `Article` |
| 下書き | `Draft` |
| 公開済み | `Published` |
| タイトル | `Title` |
| 本文 | `Body` |
| 公開識別子 | `PublicArticleId` |
| 外部投稿/同期 | `CrossPost` / `Sync` |
