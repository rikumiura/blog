
# ブログサイト SUDOモデリング定義書

## 1. S：システム関連図 (System Context Diagram)
システムの境界と外部との関わりを定義します。

*   **アクター**
    *   **執筆者（管理者）**: あなた一人。ログイン認証を経て記事・タグを管理する。
*   **外部システム**
    *   **Cloudflare R2**: 記事本文（Markdown）をファイルとして保存するオブジェクトストレージ。
    *   **Cloudflare D1**: 記事・タグのメタデータを保持する SQLite ベースのデータベース。
    *   **Zenn**: API を通じて記事の投稿・同期を行う（将来連携）。
*   **インフラ**
    *   **Cloudflare Workers**: バックエンド（Hono API）のランタイム。
*   **システム境界（スコープ）**
    *   管理者認証（ログイン）。
    *   記事の執筆・編集・下書き保存・予約投稿・即時公開・削除。
    *   タグの管理（記事への紐付け・フィルタリング）。
    *   Zenn への自動投稿・同期は現時点では含まない（将来拡張）。

## 2. U：ユースケース図 (Use Case Diagram)
ユーザーの要求に対するシステムの振る舞いと、現在の注力範囲を定義します。

*   **認証**
    *   管理者としてログインする。
*   **記事管理（実装済み）**
    *   記事を新規作成する（下書きとして保存）。
    *   記事を編集する（タイトル・本文）。
    *   記事をすぐに公開する。
    *   記事を予約投稿する（指定日時に自動公開）。
    *   予約をキャンセルして下書きに戻す。
    *   記事を削除する。
    *   記事一覧をタグでフィルタリングして表示する（ページネーション付き）。
    *   記事の本文を Markdown で編集し、リアルタイムプレビューまたは分割表示で確認する。
*   **タグ管理（実装済み）**
    *   タグを作成・記事に紐付ける。
    *   記事に紐づくタグを更新する。
*   **今後の拡張ユースケース**
    *   公開済み記事を Zenn へ投稿（同期）する。
    *   外部サイト（Zenn）での公開 URL を管理する。
*   **対象外（現時点）**
    *   読者からのコメント管理。
    *   SNS への更新通知。

## 3. D & O：ドメインモデル図 & オブジェクト図
ビジネスルールと具体的なデータの姿を定義します。

### ドメインモデル図（責務の整理）

*   **集約**
    *   `Article`（記事）— 執筆から公開・予約投稿までを管理する中心集約。
    *   `Tag`（タグ）— 記事を分類するためのラベル集約。
*   **集約間の関係**
    *   `Article` と `Tag` は多対多（中間テーブル `article_tags` で管理）。
*   **この文書の責務**: システム境界・ユースケース・用語の定義。
*   **ドメイン詳細の正本**: [`aggregation.md`](./aggregation.md)
    *   `Article` 集約・`Tag` 集約の構成要素
    *   不変条件（タイトル必須・状態遷移・公開日時など）
    *   `Body` と Cloudflare R2 の保存責務分離

## 4. ユビキタス言語（対訳辞書）
実装時の表記揺れを防ぐための用語定義です。

| 日本語名 | 英語名（コード上の表記） |
| :--- | :--- |
| 記事 | `Article` |
| 下書き | `Draft` / `DraftArticle` |
| 予約済み | `Scheduled` / `ScheduledArticle` |
| 公開済み | `Published` / `PublishedArticle` |
| タイトル | `Title` |
| 本文 | `Body` |
| 本文キー | `BodyKey` |
| 公開識別子 | `PublicArticleId` |
| 予約日時 | `scheduledAt` |
| タグ | `Tag` |
| タグ名 | `TagName` |
| 管理者認証 | `authenticateAdmin` |
| 外部投稿/同期 | `CrossPost` / `Sync` |
